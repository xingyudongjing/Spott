# A.文档定位和核心功能目录

[normal] 目标：Spott V1 的全功能规则，作为产品、设计、开发、测试、运营和合规沟通的统一基础。

[normal] 原则：先写完整产品能力，再通过后台开关和运营阶段控制开放范围；所有积分价格、奖励、免费额度和风控阈值均必须后台可配置。

[normal] 术语：“局头”指活动组织者；“用户”指参与者；“群主”指群组所有者；同一自然人不需要创建 B/C 两套账号。

# B.用户角色和权限

[normal] 身份合并：账号不区分 B 端和 C 端。任何已验证日本手机号的用户都可以报名活动，也可以成为局头发布活动。

<!-- TABLE 3: 8 rows x 3 cols -->

| --- | --- | --- |
| 访客 | 浏览首页、搜索、活动详情、群组和公开主页 | 不能报名、收藏、评论、发布或消耗积分 |
| 注册用户 | 完善资料、收藏、关注、领取积分、设置通知 | 未绑定日本手机号不能报名或发布 |
| 已验证用户 | 报名、签到、评价、创建群组、发布活动 | 受积分余额、风控和账号状态限制 |
| 局头/群主 | 管理活动、报名名单、签到、群组、管理员和公告 | 不能查看非必要敏感信息；必须遵守活动规则 |
| 认证局头-初期不要 | 获得认证标识和更高发布额度 | 认证不代表平台为活动质量或收费担保 |
| 品牌账号-初期不要 | 品牌主页、品牌活动、数据和推广工具 | 需要平台人工审核与独立协议 |
| 平台运营 | 审核、下架、配置、积分调整、客服和数据查看 | 按岗位最小权限并保留审计日志 |

[normal] 账号状态：正常、限制发布、限制报名、限制积分、临时冻结、永久封禁、注销中、已注销。限制可以单独组合，不默认一刀切封禁。

[normal] 权限优先级：平台安全限制 > 账号状态 > 角色权限 > 群组/活动内权限 > 积分余额。

# C.账号系统

## C1.注册和登录

[normal] 浏览原则：浏览公开内容不需要登录；收藏、关注、报名、评论、评价、创建群组和发布活动时再触发注册。

[normal] 支持 Apple、Google 和邮箱验证码登录；同一邮箱或第三方账号需要支持账号合并。

[normal] 首次登录只要求昵称、头像和同意服务条款/隐私政策，不要求一次填完全部资料。

[normal] 登录成功后生成用户 ID；手机号、邮箱和第三方登录标识不可作为公开 ID。

[normal] 同一设备短时间创建多个账号、频繁切换账号或异常验证码请求进入风控。

[normal] 提供退出登录、全部设备退出、账号注销和注销冷静期。

## C2.日本手机号验证

[normal] 触发时机：只在用户首次报名活动、首次发布活动或首次创建群组时触发，不打断纯浏览流程。

[normal] 仅接受可以接收短信的日本 +81 手机号；发送 6 位验证码，10 分钟有效。

[normal] 同一手机号原则上只能绑定一个有效账号；账号合并或换号需要验证原手机号，无法验证时进入人工申诉。

[normal] 验证码连续错误 5 次后暂停 30 分钟；单手机号、设备和 IP 均设置发送频率上限。

[normal] 验证成功一次性发放 500 免费积分；重复解绑、换号或账号合并不重复奖励。

[normal] 未完成验证时，报名、发布和创建群组按钮显示原因与继续验证入口，不使用笼统报错。

## C4.隐私和注销

[normal] 用户可以查看和修改营销通知、位置权限、公开资料和拉黑名单。

[normal] 账号注销前提示未结束活动、未处理投诉、群主转让和积分余额；付费积分处理规则需要在购买前明确。

[normal] 注销申请进入 14 天冷静期；期间登录可撤销。冷静期后执行匿名化/删除，法律要求保留的日志按期限保存。

[normal] Spott 与 Stareal 数据逻辑分区；只有用户主动选择接收房产或生活服务信息时才进入相应营销流程。

### 验收标准

[normal] 访客可以完成首页、搜索和详情浏览，不出现强制登录墙。

[normal] 未绑定手机号的注册用户可以收藏和关注，但报名、发布和创建群组时必须进入验证流程。

[normal] 手机号重复绑定、验证码超限、换号和注销均有明确状态和人工处理入口。

# D.首页发现、搜索和收藏

## D1.首页结构

[normal] 目标：用户打开后 10 秒内看见与自己时间、地区和兴趣相关的真实活动；首屏不以平台宣传代替活动供给。

[normal] 顶部：当前地区、搜索入口、通知入口和个人头像。

[normal] 推荐模块：今日可参加、本周末、附近热门、兴趣推荐、新活动、认证局头、关注群组更新。

[normal] 活动分类：亲子、户外、运动、城市探索、美食、游戏、文化艺术、技能学习、职业交流、其他。

[normal] 运营位：最多 1 个首屏 Banner，必须标注推广/运营推荐，不能连续挤压自然活动列表。

[normal] 地区默认根据用户选择而不是强制精确定位；允许日本全国切换都道府县和主要城市。

## D3.搜索和筛选

[normal] 搜索对象包括活动标题、活动说明、标签、局头昵称和群组名称。

[normal] 筛选条件包括日期、都道府县/城市、距离、分类、费用、语言、人数规模、认证局头和是否有名额。

[normal] 排序包括推荐、距离、时间、最新发布和即将满员。

[normal] 搜索历史只保存在用户账号内，支持单条删除和全部清除。

## D4.收藏和关注

[normal] 收藏：登录用户可以收藏活动；收藏不占名额，活动即将开始或名额变化时按用户通知设置提醒。

[normal] 关注：用户可以关注局头和群组；关注内容进入首页专属流，取消关注不需要二次确认。

[normal] 去重：同一活动只记录一次收藏；重复点击为取消收藏，不重复发放积分。

### 验收标准

[normal] 地区、日期、分类和费用筛选可以组合，返回结果与条件一致。

[normal] 活动已满、取消、结束或下架时，卡片和详情状态一致。

[normal] 分享链接打开活动详情不要求登录，并能正确归因来源。

# E.活动发布系统--B端（局头）

[normal] 产品目标：用最简洁的操作和步骤完成活动发布，同时在发布前收集履约、安全和收费边界所需信息。

[normal] 发布条件：账号正常、已绑定日本手机号、积分足够、未超过发布频率且通过必要审核。

## E1.发布流程

[normal] 选择活动类型和分类；可从历史活动复制。

[normal] 填写标题、封面、活动说明、时间、地区和集合方式。

[normal] 设置人数、报名截止、候补、报名问题和参与限制。

[normal] 选择免费或收费；收费活动填写金额、App 外收款方式、收款主体和取消退款规则。

[normal] 设置签到方式、关联群组、评论权限和分享海报。

[normal] 预览完整详情，系统检查缺失项和风险词。

[normal] 确认消耗积分后提交；普通活动进入自动+抽样审核，高风险活动进入人工审核。

## E4.局头工作台

[normal] 收费活动未填写收款主体与退款规则时不能提交。

[normal] 关键字段变更、取消、下架和审核拒绝都有明确通知、积分处理和审计记录。

# F.报名活动系统--C端（用户）

## F1.活动详情

[normal] 未登录浏览：活动详情、局头公开资料、收费信息和风险提示均可浏览；报名时再登录并验证日本手机号。

[normal] 展示标题、图片、时间、城市/区域、集合信息、人数、费用、报名截止、活动说明、注意事项、局头和群组。

[normal] 收费活动固定展示“费用由组织者自行收取，Spott 不经手活动款”。

[normal] 精确地址可以由局头设置为报名确认后显示；未报名用户只看城市/区域和集合范围。

[normal] 局头信誉优先展示已完成活动数、到场率、连续组织和有效投诉情况，不显示充值金额排名。

[normal] 报名按钮根据状态显示：立即报名、申请参加、加入候补、已满、报名截止、活动取消或活动结束。

## F2.报名流程

[normal] 登录并完成日本手机号验证。

[normal] 确认参与人数量和局头设置的报名问题。

[normal] 阅读活动规则、收费边界、取消规则和风险提示。

[normal] 收费活动按局头说明在 App 外付款；Spott 仅记录局头自报或用户确认的状态，不处理资金。

[normal] 确认消耗报名积分；自动通过活动直接占位，审核制活动进入待确认。

[normal] 报名成功后显示日历添加、路线、分享、联系局头的公开方式和加入群组入口。
