### 5.1 待你决策的 P0：运营后台 MFA（`fix/ops-mfa` 分支，未合并）

- 分支已实现**真 TOTP**（RFC 6238，替代 main 上"永远返回 true 的假 MFA"）。

- **需你拍板**：初始超管的 TOTP 怎么引导？（a）超管带外发放密钥（账号创建时预置）（b）一次性注册令牌（c）已完成 MFA 的超管审批新管理员。定了我来收口。

### 5.2 结构性阻塞的成就（不硬做，避免引入更糟问题）

- **分类徽章**（城市探索者 / 兴趣专长 / 主题专长）：`categoryCheckins` 用 **category_id（UUID）** 作 key，而分类**没有稳定 slug 或确定性 UUID 种子**。硬种一个写死的 category UUID，换环境就失效。需先建**分类 slug 机制**（schema + 设计决策）才能安全播种。
- **可信局头**：依赖"认证局头"，而 V1 明确"初期不要"认证。`certified` 修正为 false 后，此徽章正确地暂不可获得 —— 这是**符合 V1 范围**的，不是缺陷。

### 5.3 已保住、待上线接入的分支

- **`fix/otp-delivery`**：OTP 投递抽象层，把 `OTP_PROVIDER=primary` 的**静默成功**改为 fail-closed，内含 **AWS End User Messaging Notify 适配器**（`@aws-sdk/client-pinpoint-sms-voice-v2`）。SMS 上线时接入即可（你已决定 dev 阶段用 `console`，验证码打到服务端日志）。

## 7. 与"全面超过 Luma 与 Meetup"的差距

**诚实结论：差距已显著缩小，但尚无证据支持"已超越"。**

**本轮已补上的竞品核心：**
- **票务壳**（Luma 对标）：多票种、票档、优惠码、嘉宾名单 —— 但**收款仍在 App 外，Spott 只记录状态**（守日本资金決済法边界）。
- **受控消息**（Meetup 对标）：群组讨论区 —— 但**不做陌生人私信**（守安全/未成年人边界）。

**Spott 已经可能形成优势的（可验证）：** 90 秒轮换动态签到码（两家都是静态码）、签到纠错审批链、不可变审计日志、幂等双分录积分账本、精确地址加密分级可见、日本本地化 + 中日英三语。

**仍缺 / 需决策的对标能力：** Stripe 在线票务与结算（与"不代收"根本冲突，需产品+法律决策）、周期活动、Calendar 组织中枢、多主办方、付费会员订阅、公开 API/Webhook、相册、Meetup Pro 组织网络、Android。这些多为**产品定位差异或外部前置**，不是简单工期欠账。

## 8. 范围决策（2026-07-17 所有者拍板，不可违反）

1. **票务壳，不代收**：做全部票务*功能*，收款由组织者在 App 外完成，Spott 只记状态。代码/账本/文案**禁止**出现：平台收款、代收、代退、积分抵活动费、局头提现、按活动费抽成、Stripe 等真实支付。
2. **受控消息，不做开放私信**：做群组讨论区、活动聊天室（仅报名者、结束归档）、相册。陌生人 1v1 私信保持关闭；消息入口必须绑定群成员或活动报名者身份。
3. **不做 Android**：集中 iOS + Web；后端 API/ID/通知模型继续保持不阻塞未来 Android 接入。
4. **施工顺序**：先保护合并并行工作 → 再修 P0 → 最后加竞品功能（本轮已按此执行）。

## 9. 工程注意事项（下一位开发者必读的坑）

- **Node 版本**：Web/所有测试**必须**用 Node 24（`export PATH=/opt/homebrew/opt/node@24/bin:$PATH`）。系统默认 Node 26 会启用实验 WebStorage 干扰 jsdom，导致**假失败**。
- **两个门禁都要跑**：`pnpm check` **不含**迁移 manifest / 集成覆盖 / 工具链锁校验（那些在 `node --test tests/quality/*.test.mjs`）。改迁移或加 `*.integration.spec.ts` 后两个都要过。
- **新增 integration spec 要登记**：`tests/quality/integration-coverage.json`，否则 `test:quality:ci` 红。
- **迁移严格连续**：编号必须 `sequence === index+1`，文件名前缀=sequence，sha256 是**内容哈希**（改名不变）。并行分支各自加迁移会撞号，合并时需重编号（本轮 0024–0028 均为合并时收敛）。
- **本地假失败源**：`apps/web/.env.local`（被 gitignore）会覆盖 `NEXT_PUBLIC_SITE_URL`；未提交的 `pnpm-lock.yaml` churn 会让工具链锁校验红。跑门禁前确认工作区干净。
- **Git 身份**：用 `xingyudongjing`，不要用 `yaokai4`。提交用精确路径，**不要 `git add -A`**。
- **破坏性操作禁令**：严禁 `git reset --hard` / `git clean -fd` / 未经确认的全量 stash / 删 `.worktrees/`。
- **修复纪律**：改任何 bug 前先写红测试并**证明它在修复前失败**；撤销修复验证"测试有牙齿"时，一定 grep 确认代码真变了（曾有 sed/perl 未匹配导致"撤销后仍通过"的假象）。
