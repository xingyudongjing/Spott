## 0. 北极星目标

**用一句话定义成功：Spott 要成为 Luma 与 Meetup 的"合体升级版"——在功能覆盖上不弱于两者，在产品体验、精致度与高级感上明显超越两者，并在日本本地化、信任安全与跨语言体验上形成他们没有的护城河。**

这个目标拆成四条可衡量的维度，本文围绕它们组织：

| 维度 | 含义 | 衡量方式 |
| --- | --- | --- |
| **功能对等/超越** | Luma 的活动+票务体验、Meetup 的社群+成员关系，Spott 都要有对应或更强的能力 | 第 4 节"竞品超越矩阵"逐条对照 |
| **产品体验（UX）** | 发现→报名→到场→复购的闭环顺滑、状态清晰、下一步明确，弱网/离线/异常不掉链子 | 第 5 节 UX 标准 + 产品文档 F/R 验收 |
| **精致度 / 高级感（视觉）** | "东京余光 Tokyo Afterglow"设计语言：像一本会动的城市活动杂志，而不是票务后台或社交信息流 | 第 6 节视觉卓越标准 + 真机/Figma 验收 |
| **护城河（差异化）** | 日本同城语境、中日英三语、真实履约信任体系、双账本积分、跨端可恢复 | 第 4.3 节"已成型的优势" |

> **诚实前提**：本文把"超越"当作**方向和路线**，不当作**已达成的事实**。当前尚无用户研究、留存、性能、无障碍数据证明"已超越"。任何对外表述都应说"目标是超越"，不说"已超越"。

### 4.3 已成型的、Luma/Meetup 没有的护城河（可验证的差异化）

- **日本同城语境 + 中日英三语**：数据结构分 `title_default` 与翻译表，locale API 格式化。
- **真实履约信任体系**：90 秒动态签到码、补签纠错审批、真实到场率区间、不可变审计日志。
- **双分录积分账本**：幂等、可冲正、失败冻结、不可静默改余额（本轮修复了退款白拿/无限刷两个漏洞）。
- **精确地址加密分级可见**：服务端字段级授权，仅 confirmed/checked_in 且未被安全隐藏时返回。
- **跨端可恢复**：草稿 baseVersion 三方合并、离线操作队列、幂等重放。

> **结论**：功能层面，Spott 已"覆盖 Luma 90% 的组织者体验 + Meetup 的社群粘性"，且在信任/签到/账本/隐私上**已经强于**两者。真正的"超越"卡在三处产品决策（支付、私信/会员、周期/Calendar 模型）和一轮体验/视觉打磨上。

## 12. 工程注意事项（下一位开发者必读）

- **Node 24 强制**：`export PATH=/opt/homebrew/opt/node@24/bin:$PATH`。系统默认 Node 26 会启用实验 WebStorage 干扰 jsdom → 假失败。
- **两个门禁都要跑**：`pnpm check` **不含** 迁移 manifest / 集成覆盖 / 工具链锁（在 `node --test tests/quality/*.test.mjs`）。
- **新 integration spec 要登记** `tests/quality/integration-coverage.json`。
- **迁移严格连续**：`sequence===index+1`，文件名前缀=sequence，sha256 是内容哈希；并行分支加迁移会撞号，合并时重编号。
- **本地假失败源**：`apps/web/.env.local` 覆盖 `NEXT_PUBLIC_SITE_URL`；未提交 `pnpm-lock.yaml` churn。跑门禁前确认工作区干净。
- **Git**：身份用 `xingyudongjing`；精确路径提交，不 `git add -A`；严禁 `git reset --hard`/`git clean -fd`/删 `.worktrees/`。
- **修复纪律**：改 bug 前先写红测试并证明修复前失败；撤销验证"测试有牙齿"时 grep 确认代码真变了。
