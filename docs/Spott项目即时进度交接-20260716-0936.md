### D. iOS 会话隔离独立审查修复

`30951f7` 定向 37/37 通过，但独立审查不是 clean，发现 3 个 P1：

1. API 请求未绑定发起时会话，A 的延迟 200/401 可能在 B 激活后被接受，甚至用 B 重试 A 的旧 POST。
2. 同步队列没有持久化 owner scope，冷启动时孤儿 A pending operation 存在被 B flush 的风险。
3. 首次 sensitive reset 失败会让后续 reset 永久短路，上层又用 `try?` 吞掉错误。

已指定专门修复任务：先为上述三条各写 RED，再实现 auth epoch/session 验证、持久化 queue owner 与可重试/fail-closed reset，完成后重跑全量 iOS 测试和 Debug/Release。
